## Documented Gaps

### Gap-1 (Phase 1): CR-04 — Silent naive→UTC coercion in storage layer
- **Severity:** Medium — data correctness
- **Finding:** `ts_utc` validation in Bar model doesn't catch naive timestamps already in UTC numeric range; storage layer may silently coerce them.
- **Impact:** Bars stored with incorrect UTC offset could silently corrupt backtest windows, violating "trust the numbers."
- **Deferred:** Acceptable for paper trading. Fix before any live use.

### Gap-2 (Phase 1): CR-01 — data_hash dtype fragility (FND-08)
- **Severity:** Low — reproducibility
- **Finding:** data_hash computed over DataFrame dtype that can vary across pandas/pyarrow versions; same input could produce different hashes on different installs.
- **Impact:** CI reproducibility gate may produce false negatives on version upgrades.
- **Deferred:** Low probability in practice given pinned deps. Log in tech debt backlog.

### Gap-3 (Phase 1): CR-03 — Decimal-as-decoration in Bar model
- **Severity:** Low
- **Finding:** Bar model uses Decimal for price fields but some code paths convert to float before persistence; Decimal precision not enforced end-to-end.
- **Deferred:** Acceptable for paper trading at 1m ES resolution.

### Gap-4 (Phase 1): CR-02 — trading_days dual-contract edge case
- **Severity:** Low
- **Finding:** trading_days() returns ambiguous results at contract rollover boundaries in dual-contract windows.
- **Deferred:** Does not affect single-day backtest runs. Note in walk-forward config docs.

### Gap-5 (Phase 1): WR-06 — RolloverDetector applied to SPY
- **Severity:** Info — logic mismatch
- **Finding:** RolloverDetector uses CME quarterly cycle logic but is being applied to SPY bars (SPY has no futures rollover). Rollover-seam flag will never fire for SPY, which is correct behavior — but the detector is wasted.
- **Deferred:** Non-issue for function. Clean up when ES-native feed replaces SPY proxy.

### Gap-6 (Phase 3): BL-1 — Win-rate test asserts ≤90% instead of 40–60% band
- **Severity:** Low — test coverage gap
- **Finding:** BL-1 win rate acceptance test uses `win_rate <= 0.9` (loose bound) instead of the ROADMAP's stated `40–60%` band. Test doesn't verify healthy win rate; it only catches degenerate cases.
- **Deferred:** Strategy behavior is correct (Phase 5 UAT showed realistic trade behavior). Tighten assertion when real ES historical data is used.

### Gap-7 (Phase 6): TV-02 — ORB box drawn with stub coordinates
- **Severity:** Medium — visual accuracy
- **Finding:** TVBridge draws the ORB box using `signal.entry × 1.001` and `signal.entry × 0.999` as high/low instead of the actual session opening-range H/L extracted from bar history.
- **Impact:** ORB box shown on TradingView chart will be visually wrong (tiny ±0.1% band around entry, not the actual ORB zone). Does not affect trade logic.
- **Fix required:** TVBridge needs access to the ORB high/low from StrategyContext when the signal fires. Signal model (Phase 2) should carry `orb_high` and `orb_low` fields, or TVBridge subscribes to bar events to reconstruct the window.
- **Deferred:** Phase 6 checkpoint was auto-approved. Must fix before TradingView integration is considered reliable for manual trade review.

### Gap-8 (Housekeeping): REQUIREMENTS.md traceability table stale
- **Severity:** Info — documentation
- **Finding:** Traceability table in REQUIREMENTS.md shows "Pending" for all Phase 2–8 requirements. Checkboxes for STR-01..06, OPT-01..09, SP-02..06, RM-01..08, UI-02/03/05/06/07/09, TV-01/04/05 are unchecked despite being implemented.
- **Fix:** One-pass update to tick all completed requirement checkboxes and update traceability table.
- **Deferred:** Documentation only, no code impact.

### Gap-10 (Integration): POST /backtests/run is a stub — UI Run Backtest never executes BacktestEngine
- **Severity:** Medium — UX gap (feature appears to work but produces fake results)
- **Finding:** `packages/api/src/api/routes/strategies.py` `_run_backtest_task` sleeps 2s then marks the run `complete` with no trades or equity curve written. The Run Backtest UI button always shows instant fake completion.
- **Impact:** The full backtest pipeline (BT-01..09) works correctly via CLI (`run_backtest.py`); only the UI button is unwired.
- **Fix:** Wire `_run_backtest_task` to instantiate `BacktestEngine(strategy, risk_manager, executor, config)` and call `.run(bars)`. Requires resolving the DataSource dependency at request time.
- **Deferred:** Documented in Phase 7 VERIFICATION.md anti-patterns as "real engine wiring deferred to Phase 8." Phase 8 closed without addressing it — becomes first-priority v1.1 item.

### Gap-9 (Process): Phase 2 missing VERIFICATION.md
- **Severity:** Info — process gap
- **Finding:** Phase 2 (strategy-engine) has PLAN.md and SUMMARY.md for both plans, but no VERIFICATION.md. The SUMMARY shows self-check PASSED with 244 tests green. Phase 3 backtester drives ORBStrategy confirming the integration works.
- **Fix:** Retroactive VERIFICATION.md for Phase 2 would close the process gap.
- **Deferred:** Non-blocking. Evidence in SUMMARY is strong.
