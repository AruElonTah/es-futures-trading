Extend the Phase 1 strategy model stubs (Signal + StrategyContext) with full field definitions and build the four look-ahead-safe indicator implementations (ATRWilder, SessionVWAP, EMA, ADR) that ORBStrategy will consume in Plan 02-02.

The critical correctness invariant for this plan is **look-ahead safety**: `indicator.snapshot_at(t)` must equal the indicator recomputed from `bars[0:t]` exclusively. Any regression that uses the current bar's data to compute the indicator value it exposes to `on_bar` is a leakage bug (BL-2 per RESEARCH.md) that silently inflates backtest performance.

Output: A fully-typed indicator module (`trading_core.indicators`) with a consistent push/snapshot_at/current API, fully-filled Signal + StrategyContext models, and a synthetic ORB-day fixture reusable in Plan 02-02.

Indicator public API (all 4 must implement this shape):
  - push(bar: Bar) -> None: appends bar to internal history, updates stored values
  - snapshot_at(t: int) -> Decimal | None: value computed from bars[0:t]; returns None if t < warmup_bars

ATRWilder:
  - Wilder's smoothed ATR; period default 14
  - True Range: max(high-low, |high - prev_close|, |low - prev_close|)
  - First value (when history >= period): simple mean of first `period` true ranges
  - Subsequent: (prev_atr * (period-1) + tr) / period
  - Use Decimal arithmetic throughout — no float intermediate
  - warmup_bars() = period + 1 (need prev_close for TR on bar 2; need period TRs for first SMA)

SessionVWAP:
  - Resets when the calendar date (ET) changes from the previous bar's date
  - Price = (high + low + close) / 3 (typical price)
  - Accumulate: cum_tpv (cumulative typical_price * volume) and cum_vol (cumulative volume)
  - vwap = cum_tpv / cum_vol  (Decimal division)
  - warmup_bars() = 1 (valid from the first bar of a new session)
  - snapshot_at(t): value as of the start of bar t (using bars 0..t-1)

EMA:
  - period default 20; multiplier = 2 / (period + 1)
  - First EMA (when history == period): simple mean of first `period` closes
  - Subsequent: close * multiplier + prev_ema * (1 - multiplier)
  - Use Decimal arithmetic; multiplier stored as Decimal(2) / Decimal(period + 1)
  - warmup_bars() = period

ADR (Average Daily Range):
  - period default 10 (trading days)
  - Internally resamples accumulated 1m bars to daily OHLC via pandas
  - Daily range = daily_high - daily_low for each complete past trading day
  - BL-3 invariant: resample produces `daily_df`; compute rolling(period).mean() on
    the `range` column; then SHIFT by 1 day (`daily_ranges.shift(1)`) so today's
    partial range never leaks into today's ADR reading
  - ADR at bar[t] = the shifted rolling mean aligned to bars[0:t]
  - warmup_bars() = (period + 1) * 390 (approx; at least period complete days needed)
  - Implementation approach: after each push, recompute from scratch on the accumulated
    bar list (acceptable cost for Phase 2; Phase 4 can optimize if needed)
  - Alternatively, track daily state incrementally — either approach valid
