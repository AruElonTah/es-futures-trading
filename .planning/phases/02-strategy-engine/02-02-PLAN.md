<objective>
Implement `ORBStrategy` — the reference Opening Range Breakout strategy that satisfies the `Strategy` protocol defined in Phase 1, consumes the look-ahead-safe indicators from Plan 02-01, reads its configuration from `config/strategies/orb.yaml`, and emits a `Signal` object with full audit fields on a valid breakout.

Simultaneously implement `StrategyRegistry` — the thin YAML loader that enables the UI (Phase 7) to enumerate and hot-reload strategies by reading `config/strategies/*.yaml` with no Python-side hardcoded fallbacks.

The critical correctness invariants for this plan:
1. **Look-ahead safety**: ORBStrategy must use `ctx.atr` (from prior bars) for stop computation, never the current bar's ATR. The integration test in `test_indicator_leakage.py` enforces this.
2. **Rollover guard**: `on_bar` returns None immediately when `ctx.rollover_seam=True`.
3. **One signal per session**: after the first signal fires, ORBStrategy ignores subsequent breakouts in the same RTH session.
4. **Config purity**: all params (opening_range_minutes, atr_stop_mult, r_target) come from YAML via `ORBConfig`; no `or`-fallback defaults in `on_bar` logic.

Output: A working ORBStrategy that can be driven bar-by-bar through a synthetic day, produces exactly one signal on a valid breakout, and is loadable from YAML through StrategyRegistry.
</objective>

<interfaces>
ORBConfig (dataclass or Pydantic BaseModel — use dataclass for lightweight config):
  - strategy_id: str = "orb-v1"
  - strategy_version: str = "1.0"
  - opening_range_minutes: int = 15
  - atr_period: int = 14
  - atr_stop_mult: float = 1.5  # float OK here — it's a multiplier, not a price
  - r_target: float = 2.0
  - ema_period: int = 20
  - min_range_ticks: int = 2  # minimum ORB width (in ticks) to avoid tiny-range signals

ORBStrategy internal state:
  - _config: ORBConfig
  - _atr: ATRWilder (period = config.atr_period)
  - _vwap: SessionVWAP
  - _ema: EMA (period = config.ema_period)
  - _bar_count: int = 0  # cumulative bars seen (for bar_index in ctx)
  - _session_date: date | None = None  # current ET session date (for session reset)
  - _orb_high: Decimal | None  # set during opening range
  - _orb_low: Decimal | None   # set during opening range
  - _orb_bars: int = 0  # count of bars seen in current opening range
  - _signal_fired: bool = False  # True after first signal in current session

ORBStrategy.on_bar logic:
  1. If ctx.rollover_seam: return None immediately (guard before any state update)
  2. Get current ET date from bar.ts_utc
  3. If ET date changed from _session_date: reset session state (_orb_high=None, _orb_low=None, _orb_bars=0, _signal_fired=False, _session_date=new_date)
  4. If not ctx.warmup_complete: return None (insufficient indicator history)
  5. If _orb_bars < config.opening_range_minutes: update ORB high/low from bar, increment _orb_bars, return None
  6. If _signal_fired: return None (one signal per session enforced)
  7. Check for breakout: if bar.close > _orb_high AND (bar.close - bar.open) > 0 (bullish bar):
     - Validate ORB width: if (_orb_high - _orb_low) < Decimal(str(instrument.tick_size * config.min_range_ticks)): return None
     - entry = _orb_high (next bar open is approximated as ORB high in unit tests; Phase 3 fills at next bar open)
     - stop = _orb_high - (ctx.atr * Decimal(str(config.atr_stop_mult)))  (ATR-based stop)
     - target = entry + Decimal(str(config.r_target)) * (entry - stop)
     - _signal_fired = True
     - return Signal(strategy_id=..., strategy_version=..., ts_utc=bar.ts_utc, side="long", entry=entry, stop=stop, target=target, size_hint=Decimal("1"))
  8. Similarly for short breakout (bar.close < _orb_low AND bearish bar) — mirror of step 7
  9. return None

  NOTE: The strategy updates its indicators AFTER calling on_bar's logic — but since the indicators are maintained internally (not via ctx), the driver must handle the ctx construction. The correct driver pattern for look-ahead safety:
    ```
    for bar in bars:
        # 1. Build ctx from indicator snapshots BEFORE this bar
        ctx = StrategyContext(
            rollover_seam=bar.rollover_seam,
            warmup_complete=strategy.is_warm(),
            bar_index=strategy._bar_count,
            ts_utc=bar.ts_utc,
            atr=strategy._atr.current,      # from prior bars
            session_vwap=strategy._vwap.current,
            ema=strategy._ema.current,
            adr=None,  # Phase 2 leaves adr=None; ADR wired in Phase 3
        )
        # 2. Call on_bar with current bar + prior-bar context
        signal = strategy.on_bar(bar, ctx)
        # 3. Push bar to indicators AFTER on_bar
        strategy._push_bar(bar)
    ```
  ORBStrategy must expose a `_push_bar(bar: Bar) -> None` method (or equivalent) that:
    - Pushes bar to _atr, _vwap, _ema
    - Increments _bar_count
  The driver (test or backtester) calls _push_bar AFTER on_bar. This ensures look-ahead safety.

  Alternatively: ORBStrategy can expose `is_warm() -> bool` that delegates to `self._atr.is_warm`.

ORBStrategy public interface (satisfies Strategy protocol):
  - name: str = "opening_range_breakout"  (class attribute)
  - version: str = "1.0"                  (class attribute, overridden by config)

    ```python
    """Strategy registry — YAML-based loader for config/strategies/*.yaml files.

    Phase 2: supports ORBStrategy only. Phase 7 adds dynamic class dispatch via importlib.
    """
    from __future__ import annotations
    import yaml
    from pathlib import Path
    from .orb import ORBConfig, ORBStrategy


    class StrategyRegistry:
        @staticmethod
        def load(yaml_path: str | Path) -> ORBStrategy:
            path = Path(yaml_path)
            with path.open("r", encoding="utf-8") as f:
                data = yaml.safe_load(f)
            params = data.get("params", {})
            config = ORBConfig(
                strategy_id=data.get("strategy_id", "orb-v1"),
                strategy_version=str(data.get("version", "1.0")),
                **params,
            )
            return ORBStrategy(config)

        @staticmethod
        def list_strategies(strategies_dir: str | Path) -> list[str]:
            d = Path(strategies_dir)
            names = []
            for p in sorted(d.glob("*.yaml")):
                with p.open("r", encoding="utf-8") as f:
                    data = yaml.safe_load(f)
                if data and "name" in data:
                    names.append(data["name"])
            return names
    ```

    ```python
    the signal's stop distance would differ from what a correctly-prior-ATR strategy

    from trading_core.strategy.orb import ORBConfig, ORBStrategy
    from trading_core.strategy.models import StrategyContext

    def test_orb_strategy_no_lookahead():
        """The ATR used in the breakout signal was computed BEFORE the breakout bar."""
        config = ORBConfig(opening_range_minutes=15, atr_period=14, atr_stop_mult=1.5, r_target=2.0)
    ```
