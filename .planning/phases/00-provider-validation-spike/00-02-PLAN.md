MCP Python SDK API surface (from `mcp` package on PyPI, verified in RESEARCH.md):

```
from mcp import ClientSession, StdioServerParameters
from mcp.client.stdio import stdio_client
from contextlib import AsyncExitStack

server_params = StdioServerParameters(
    command="node",
    args=[r"C:\Users\Admin\tradingview-mcp-jackson\src\server.js"],
    env=None,                      # inherit parent env; the server reads CDP_URL from env if needed
)

async with AsyncExitStack() as stack:
    stdio_transport = await stack.enter_async_context(stdio_client(server_params))
    read, write = stdio_transport
    session = await stack.enter_async_context(ClientSession(read, write))
    await asyncio.wait_for(session.initialize(), timeout=15.0)

    tools_response = await session.list_tools()
    tool_names = sorted(t.name for t in tools_response.tools)

    result = await session.call_tool("tv_health_check", {})
    result = await session.call_tool("chart_set_symbol", {"symbol": "CME_MINI:ES1!"})
    result = await session.call_tool("chart_set_timeframe", {"timeframe": "1"})
    result = await session.call_tool("data_get_ohlcv", {"count": 390, "summary": False})
```

Tool-call result shape: `result.content` is a list of `TextContent` / `JSONContent` objects; the convention used by `tradingview-mcp-jackson` is to return a JSON string in the first item's `.text` field that parses to `{"success": bool, ...payload}`. The executor should extract via `result.content[0].text` and `json.loads(...)` it.

TV symbology (RESEARCH.md §"Local server confirmed shape"):
- ES continuous front-month: `"CME_MINI:ES1!"`
- 1-minute timeframe: `"1"` (string; TV uses minute counts as strings for intraday: "1", "5", "15", "60", "D")

`data_get_ohlcv` quirk (RESEARCH.md §"Critical implementation notes" #5):
- Reads currently-visible chart bars only. Does NOT take date arguments.
- To get the previous RTH session, sequence is: chart_set_symbol → chart_set_timeframe → chart_scroll_to_date (ISO 8601) → data_get_ohlcv(count=390, summary=False).
- For the smoke test, scroll to the most recent RTH close — RESEARCH.md uses `2026-05-13T20:00:00Z` (16:00 ET) as the example. The executor should compute "yesterday's 20:00Z" relative to the current date.

Critical timeouts and pitfalls (RESEARCH.md "Critical implementation notes"):
1. Wrap `session.initialize()` in `asyncio.wait_for(timeout=15.0)`; fail loudly with "TV Desktop not running" if it hangs.
2. `tv_health_check` is the FIRST real call; if it fails, abort the smoke test with a clear message — do NOT auto-launch via `tv_launch` (keep the test deterministic).
3. `stdio_client` consumes stdout for MCP protocol. Set up a separate background reader on the subprocess's stderr and tee it to `.planning/research/spike-0/tv-mcp-stderr.log`.
4. Working directory: server uses ES modules with absolute imports per package.json `"type": "module"` — cwd should not matter, but the script should `os.chdir` to the server directory as a safety belt before launching.
5. **Allowlist of tools that may be called** (RESEARCH.md Threat T-03 = "spike script accidentally calls a destructive tool"):
   - READ ONLY: `tv_health_check`, `chart_get_state`, `data_get_ohlcv`, `quote_get`, `list_tools` (implicit), `tv_list_charts` (if present)
   - REVERSIBLE WRITES: `chart_set_symbol`, `chart_set_timeframe`, `chart_scroll_to_date`
   - **FORBIDDEN**: `draw_shape`, `alert_create`, `alert_delete`, `pine_set_source`, `pine_smart_compile`, any tool starting with `replay_` (could leave a session active), any tool that mutates persistent TV state.
   The smoke script must hard-code this allowlist and assert each `call_tool` name is in the allowlist before invoking (defense in depth — even a typo cannot reach a destructive tool).
6. **Restore-on-exit** (RESEARCH.md Threat T-04): At session start, call `chart_get_state()` and store the initial symbol/timeframe. In a `try/finally`, restore on exit so the operator's chart returns to its original state.

Quote staleness check (per the pre-planning user answer — TV plan is paid with real-time CME):
- After `chart_set_symbol("CME_MINI:ES1!")` succeeds, call `quote_get(symbol="CME_MINI:ES1!")`. The result includes a timestamp.
- Compute `staleness_seconds = (now_utc - quote.timestamp_utc).total_seconds()`.
- During RTH hours: if staleness > 5s, log a WARNING in the transcript ("real-time quote staleness exceeded 5s; verify TV CME subscription"). This is defense-in-depth, NOT a blocking failure — the operator has confirmed pre-planning that the subscription is paid + real-time, so this is just a verifiable record.
- Outside RTH hours: skip the check (no live ticks).
