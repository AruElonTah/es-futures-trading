Endpoint set (RESEARCH.md §"The 4 probe calls"):
- GET https://api.twelvedata.com/stocks?symbol=ES
- GET https://api.twelvedata.com/commodities?symbol=ES
- GET https://api.twelvedata.com/etf?symbol=SPY
- GET https://api.twelvedata.com/time_series?symbol=SPY&interval=1min&outputsize=5&apikey=<KEY>

Rate-limit headers to capture on every response (RESEARCH.md §"Rate-limit headers"):
- api-credits-used
- api-credits-left

Failure-mode taxonomy (RESEARCH.md §"Failure modes to capture"):
- HTTP 200 + status="error" + code=400 + "symbol not found"  → genuine negative (ES unsupported)
- HTTP 200 + status="error" + code=401                       → bad API key (fix; do NOT commit)
- HTTP 200 + status="error" + code=403 mentioning "plan"     → endpoint requires paid tier
- HTTP 429                                                    → rate limit hit (NOT "feature unavailable")
- Empty data array on /commodities?symbol=ES                 → strong negative
- Non-empty data but exchange != "CME" / type != "Futures"   → matched a different ES (equity ticker)

API-key redaction (load-bearing — gitleaks does not yet exist in this repo):
- Replace API_KEY substring inside any URL before writing to disk: url.replace(API_KEY, "<TWELVEDATA_API_KEY>")
- Never write the raw URL to the JSON output; only the redacted form goes to disk.
- Never include the key in stderr / stdout that gets piped into a committed file.

SPY backfill math (RESEARCH.md §"SPY Backfill Rate-Limit Math"):
- 2 years × 252 trading days × 390 RTH minutes = 196,560 bars
- 5,000 bars / call (hard cap) → 40 calls
- 1 credit / symbol / request
- Free tier: 8 credits/min, 800/day → ~5 min wall-clock with pacing, 5% daily budget
- Grow ($79): 377/min, no daily cap → <1 min, BUT 1min interval may be Pro-tier-only for equities (ETF gating ambiguous — probe #4 settles this)
- Pro ($229): 1597/min, no daily cap, 1min definitively supported
