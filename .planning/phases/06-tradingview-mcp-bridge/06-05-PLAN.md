<objective>
Fix three major bugs found during Phase 6 UAT (06-HUMAN-UAT.md), all of which cause the TV bridge integration to fail silently in production:

1. **shapes.py MCP format mismatch (UAT Gap 2):** The four draw_shape arg-builder functions produce flat dicts (`{shape, price, color, line_style, line_width}`) that do not match the MCP `draw_shape` tool schema. The tool requires `{shape, point:{time,price}, overrides:json_string}` for horizontal_line and `{shape, point, point2, overrides}` for rectangle. The current format causes every draw_shape call to fail silently inside `_safe_draw_signal`'s except block — no shapes appear on the TV Desktop chart.

2. **focus() uses chart_scroll_to_date (UAT Gap 1):** TV auto-scroll continuously overrides `chart_scroll_to_date` calls, leaving the chart on the current session. Replace with `chart_set_visible_range` which accepts explicit unix timestamp bounds for the target day's RTH session window.

3. **create_alert() missing required price param (UAT Gap 3):** The MCP `alert_create` tool requires `{condition, price, message}` but `bridge.create_alert()` only passes `{condition, message}`. Every POST /tv/alerts call returns 502 because the MCP call fails.

These are production-breaking bugs. All 94 existing tests pass because they mock at the MCP layer without validating payload correctness. The fixes are surgical: shapes.py payload format, bridge.py focus/create_alert logic, TVAlertRequest model, and matching test updates.
</objective>

## Task 4 — Fix bridge.create_alert(): add required price param

**Commit:** `fix(06): bridge.create_alert() — add required price param to alert_create MCP call`

Update `bridge.create_alert()` signature and body:

```python
async def create_alert(self, condition: str, price: float, message: str) -> str | None:
    """Create a TradingView alert via alert_create MCP tool.

    Args:
        condition: Alert condition string (max 256 chars via Pydantic validation).
        price: Price level for the alert (required by MCP alert_create tool).
        message: Alert message text (max 256 chars via Pydantic validation).

    Returns:
        tv_alert_id string from alert_create response, or None on failure.
    """
    response = await self.call_tool(
        "alert_create",
        {
            "condition": str(condition)[:256],
            "price": float(price),
            "message": str(message)[:256],
        },
    )
    if response is None:
        return None
    tv_alert_id = response.get("alert_id") or response.get("id")
    if not tv_alert_id:
        _log.warning("tv_bridge.missing_alert_id", response=repr(response))
        return None
    return str(tv_alert_id)
```
