## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| backup.ps1 params -> filesystem | -DataRoot flows into path composition and file copy/delete |
| backup.ps1 -> Remove-Item | retention cleanup deletes directories recursively |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-08 | Tampering (path traversal) | backup.ps1 -DataRoot | mitigate | All paths composed via Join-Path under -DataRoot; Remove-Item operates only on Get-ChildItem results scoped to the backups subdirectory of -DataRoot, never on an arbitrary user-supplied path |
| T-08-09 | Denial of service (unintended delete) | retention Remove-Item -Recurse | mitigate | Where-Object filter restricts deletion to directories whose CreationTime is older than RetentionDays; the pipeline is scoped to DataRoot/backups, so live data/duckdb and data/parquet are never in scope |
| T-08-10 | Information disclosure (backup at rest) | data/backups/ snapshots | accept | Snapshots inherit the data/ volume's BitLocker encryption (documented in backup.md); local single-operator machine, no network exposure |
</threat_model>
