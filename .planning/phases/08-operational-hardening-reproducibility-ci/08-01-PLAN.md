<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args -> DuckDB query | `--symbol` / `--tf` flow into a SQL SELECT |
| CLI args -> filesystem | `--output-dir` / `--risk-config` / `--duckdb-path` flow into file reads/writes |
| risk.yaml -> RiskConfig | YAML content is parsed and instantiated |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Tampering (SQL injection) | replay.py bars query | mitigate | `--symbol`/`--tf` use argparse `choices=` whitelist; bars SELECT uses `?` parameterized placeholders, never f-string interpolation |
| T-08-02 | Tampering (path traversal) | replay.py `--output-dir` write | mitigate | Output CSV path is composed under `--output-dir` with a fixed `audit/{date}.csv` suffix; date comes from the bar timestamp (ISO date), not user free-text; never write into `data/logs/audit/` (D-04) |
| T-08-03 | Elevation (arbitrary code via YAML) | risk.yaml load | mitigate | Use `yaml.safe_load` only — never `yaml.load` with the default loader |
| T-08-04 | Information disclosure | structlog audit output | accept | Local single-operator paper system, no secrets in replay path; audit CSV contains only trade/risk data |
</threat_model>
