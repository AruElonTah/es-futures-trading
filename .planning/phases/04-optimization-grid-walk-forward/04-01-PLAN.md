<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| YAML file → OptSpace parser | User-authored optspace.yaml; param names and values are untrusted until validated |
| DuckDB schema → application | DDL extensions must be idempotent; IF NOT EXISTS prevents double-create errors |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01-01 | Tampering | OptSpace.validate_axes param name check | mitigate | Validate each param name against dataclasses.fields(ORBConfig) — raises ValueError on unknown names, preventing silent typos from producing garbage combos |
| T-04-01-02 | Tampering | holdout_burns quarter string | mitigate | Quarter string computed in Python with current_quarter_str() (stdlib datetime), not from user input; stored via parameterized INSERT |
| T-04-01-03 | Spoofing | opt_runs adr_hash field | accept | Hash is written by orchestrator from the ADR file content — single operator localhost, no external input path |
</threat_model>
