# Phase 7 — UI Design Contract

> Visual and interaction contract for Phase 7: Bloomberg-Density UI Polish.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Source authority:** All design decisions pre-populated from 07-CONTEXT.md (D-01 through D-21),
> 05-UI-SPEC.md (inherited design system), and codebase scan of apps/web/. No new design
> decisions were required from the user — all 21 CONTEXT.md decisions are locked.
> This contract extends the established Bloomberg-terminal dark-monospace theme into four
> new surfaces: the 4-pane terminal layout, the Trade History pane, the Strategy Controls pane,
> and the upgraded WebSocket reconnect UX.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (vanilla Tailwind v3.4 + inline styles) — source: 05-UI-SPEC.md |
| Icon library | none (text labels and Unicode glyphs only) — source: 05-UI-SPEC.md |
| Font | CSS system monospace stack via `font-mono` Tailwind class (`body className="font-mono"` on root layout) — source: apps/web/app/layout.tsx |

**No shadcn.** This project does not use a component library. All UI is hand-authored with
Tailwind v3 utility classes and inline `style` props, consistent with Phase 3–6. Do not
introduce shadcn or any component library in Phase 7.

**New third-party dependency:** `react-resizable-panels` (~12kB) is the only new package
added in Phase 7. See Registry Safety section.

Inherited from 05-UI-SPEC.md without change. All text is monospace.

| Role | Size | Weight | Line Height | Notes |
|------|------|--------|-------------|-------|

equity curve values) use `tabular-nums` CSS feature (`className="tabular-nums"`).
This is an established pattern from ETClock.tsx and must be applied to all Phase 7
numeric columns.

scroll-target row highlight.

pane title bar background.

(and any future non-destructive button borders where the full accent `#4a90d9` is too

subdued variant to create visual depth without overemphasis.

`#ef4444` so that the histogram does not visually compete with the equity curve line at
zero-crossings. This is the only opacity usage in the design system.

## Interaction Contracts

### Pane Resize Contract

- `react-resizable-panels` manages all drag interactions. No custom pointer-event code.
- `onLayout` fires after each resize with the current size percentages array.
- Persistence: `localStorage.setItem('es-terminal-layout', JSON.stringify(sizes))`.
- On mount: read `localStorage.getItem('es-terminal-layout')`, try JSON.parse, apply to
  `defaultSize` props. Any parse error silently falls back to defaults: `[60, 40]` for
  horizontal; `[30, 40, 30]` for vertical right column.
- Min sizes enforce the dimensional constraints listed in Spacing Scale above.

### Chart Scroll-to-Trade Contract (D-12)

- Trade row click → `setFocusedBarTs(trade.entry_ts_utc)` (Zustand `WsStore`).
- `Chart.tsx` has a second `useEffect` watching `useWsStore(s => s.focusedBarTs)`.
- When non-null: find the bar index matching `focusedBarTs` in the current bars array →
  `chart.timeScale().scrollToPosition(targetIndex - visibleBarsHalf, false)` to center
  the bar in view.
- After scroll: `setFocusedBarTs(null)` to reset.
- Focused row visual: `border: 1px solid #4a90d9`, `backgroundColor: #1a1a2e`. Cleared
  immediately after scroll fires (on the next render after `setFocusedBarTs(null)`).

### Background Backtest Polling Contract (D-15)

- `POST /backtests/run` → response `{ run_id: string }`.
- `useStrategyRun()` hook polls `GET /backtests/{run_id}` every 2s while `status === "pending"`.
- Trade History pane observes polling state via the query; shows spinner while pending.
- On `status !== "pending"`: stop polling → `invalidateQueries(['backtests'])` →
  Trade History re-fetches and renders result rows.
- Only one background job at a time. While polling, the Run Backtest button is disabled.

### Strategy Hot-Reload Contract (D-14)

- `PUT /strategies/{id}/params` → 200 on success, 422 with `{ detail: string }` on validation failure.
- On 200: show inline confirmation for 2s → invalidate `['strategies']` query.
- On 422: display `detail` string below the offending field in `#f87171`.
- No optimistic param update — wait for server 200 before reflecting changes.
- Hot-reload is transparent to the user (no modal, no page reload) — the only signal is
  the `Params saved — engine reloading` transient text.

### Toggle Strategy Contract

- `POST /strategies/{id}/toggle` → 200 with updated `{ enabled: boolean }`.
- Optimistic badge update on click (immediate); confirmed on query invalidation.
- On API error: revert badge to previous state + show inline error
  `Failed to toggle strategy. Is the API running?` in `#f87171` for 3s.

### WebSocket Reconnect Contract (D-18/D-19/D-20)

- Backoff delay: `min(2^attempt × 1000ms, 30000ms) + Math.random() × 1000ms`.
- `ConnectionStatus` shows yellow (>10s since last bar) or red (>30s) during reconnect.
- No separate reconnect banner — existing `ConnectionStatus` is sufficient.
- Gap detection: if incoming message `seq > lastSeq + 1` → call `invalidateQueries` on
  positions, backtests, trades, equity. Silent to the user unless REST calls fail.
- If REST resync fails: surfaces via `DegradationBanner` (existing component).

### Hotkey Guard Rules (inherited from Phase 5, unchanged)

- `F`, `K`, `P`: skip handler when `document.activeElement.tagName` is `INPUT`, `TEXTAREA`, or `SELECT`.
- `?`: no tag guard — fires even inside inputs.
- Re-entrant guard: if a confirmation dialog is already open, pressing K/F has no effect.
- Phase 7 adds no new hotkeys. Collision detection throws at startup (existing behavior).

## Accessibility Contract

Inherited from 05-UI-SPEC.md. Phase 7 additions:

- `react-resizable-panels` `PanelResizeHandle` components receive `aria-label` values:
  `aria-label="Resize chart and side panels"` (horizontal handle),
  `aria-label="Resize blotter and history panels"` (upper vertical handle),
  `aria-label="Resize history and controls panels"` (lower vertical handle).
- Trade History table uses `<table>`, `<thead>`, `<tbody>`, `<th scope="col">` — semantic HTML.
- Strategy Controls param inputs: each `<input>` has an associated `<label>` with `htmlFor` pointing to the input `id`.
- Toggle buttons carry `aria-pressed={enabled}` boolean to announce toggle state.
- Run Backtest button carries `aria-disabled={isRunning}` while polling.
- Backtest spinner row carries `role="status"` and `aria-live="polite"`.
- Trade row click (scroll-to-chart): `<tr>` carries `role="button"` and `tabIndex={0}` so keyboard users can trigger scroll with `Enter`.
- Equity chart: `aria-label="Equity curve and drawdown chart"` on the chart container div.
