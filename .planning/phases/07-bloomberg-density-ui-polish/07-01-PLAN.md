    Read ws.py in full first. Locate ConnectionManager.__init__. Add self._seq: int = 0.

    In _subscribe_topic (the async for loop that dispatches to self._clients): before constructing msg, do:
      self._seq += 1
    For dict events: build payload = dict(event); payload['seq'] = self._seq; msg = json.dumps(payload)
    For non-dict events (model objects): build envelope = {"type": event.topic, "seq": self._seq, "payload": event.model_dump(mode="json")}; msg = json.dumps(envelope)

    This matches the pattern documented in 07-PATTERNS.md §Pattern 3. The counter is per-ConnectionManager-instance (per server start), which is per-connection semantically since each client connection gets the same shared manager. The seq is monotonic across all topics — a single counter for all messages sent from this server instance.

    Do NOT reset seq on client reconnect — the gap detection logic on the client side relies on the counter never going backward.
