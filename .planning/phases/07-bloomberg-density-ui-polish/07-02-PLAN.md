## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→PUT /strategies/{id}/params | User-supplied strategy_id used in filesystem path construction |
| client→PUT /strategies/{id}/params body | User-supplied param values written to YAML |
| client→WS | Server seq counter prevents client from faking sequence numbers |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-02-01 | Tampering | PUT /strategies/{id}/params — path traversal via strategy_id | mitigate | Validate strategy_id matches ^[a-z0-9_-]+$ before path construction; Path.resolve() + relative_to() guard (per RESEARCH.md security domain) |
| T-07-02-02 | Tampering | PUT /strategies/{id}/params — YAML injection via param values | mitigate | Pydantic ORBConfigUpdate validates all param fields before yaml.dump(); only write model_dump() output, never raw request body to YAML |
| T-07-02-03 | Spoofing | CORS expansion to include PUT | accept | PUT added to allow_methods; origins remain localhost:3000 and 127.0.0.1:3000 only — no wildcard widening |
| T-07-02-04 | Tampering | localStorage layout poisoning | mitigate | loadSizes() wraps JSON.parse in try/catch; any error silently falls back to DEFAULT_H_SIZES/DEFAULT_V_SIZES (D-06) |
</threat_model>
