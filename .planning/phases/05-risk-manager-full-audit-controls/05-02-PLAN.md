Sizing formula (RM-01 unit test requirement):
- size_for_stop(risk_dollars=1000, stop_ticks=5, instrument=MES) == 40
  Proof: floor(1000 / (5 * 5.00)) = floor(1000 / 25) = 40
  Uses instrument.point_value (NOT tick_value — "stop_ticks" in RM-01 is "stop in index points")
- size_for_stop(risk_dollars=1000, stop_ticks=5, instrument=ES) == 4
  Proof: floor(1000 / (5 * 50.00)) = floor(1000 / 250) = 4

    RM-06 — max_contracts cap:
    - size_for_stop(Decimal("100000"), Decimal("1"), MES) uncapped would be 20000,
      but min(20000, config.max_contracts=2) returns 2
    - Cap is applied AFTER size_for_stop math, not inside it

    RM-02 — all three DrawdownModel variants tracked side-by-side:
    - STATIC: hwm never decreases; floor = hwm - daily_dd_limit (set once at session start)
    - TRAILING_EOD: hwm updates at session close only (not intraday); floor = hwm - daily_dd_limit
    - TRAILING_INTRADAY: hwm updates on every fill or equity tick; floor = hwm - daily_dd_limit
    - All three HWM values written to risk_state on every check() call

    RM-03 — worst_case_loss check:
    - worst_case_loss = abs(signal.entry - signal.stop) * instrument.point_value * proposed_size
    - If (active_dd_model_floor - current_equity) < worst_case_loss: reject with reason='dd_floor_violation'
    - Active DD model determined by config.drawdown_model

    RM-04 — daily-DD circuit breaker:
    - If (realized_pnl_today + open_exposure_dollars) <= -daily_dd_limit: reject with reason='daily_dd_breaker'
    - Existing positions NOT closed (kill-switch behavior, not flatten)

    RM-08 — per-strategy concurrency cap (v1 = 1 active position per strategy_id):
    - Second signal from same strategy_id while first position is open: reject with reason='concurrency_cap'
    - FullRiskManager maintains _active_positions: set[str] (strategy_ids with open positions)
    - record_position_open(strategy_id) and record_position_closed(strategy_id) methods to update the set

    Signal approved path:
    - All checks pass → approved=True, reason='approved', adjusted_size=min(sized, max_contracts)
    - write_risk_state() called with all 12 DD model columns
    - write_audit_event() called with topic=TOPIC_RISK_DECISIONS, reason_code='approved' or rejection reason

    Module-level pure function `size_for_stop`:
    ```
    def size_for_stop(
        risk_dollars: Decimal,
        stop_ticks: Decimal,   # "stop_ticks" = stop distance in index points (see RM-01 note)
        instrument: Instrument,
    ) -> int:
        return math.floor(risk_dollars / (stop_ticks * instrument.point_value))
    ```

    Import `math` (stdlib), `Decimal` (stdlib), `Instrument` from `trading_core.instruments`,
    `get as get_instrument` from `trading_core.instruments`,
    `DrawdownModel`, `RiskConfig`, `RiskDecision`, `RiskState` from `trading_core.risk.models`,
    `Signal` from `trading_core.strategy.models`,

    `get_logger` from `trading_core.logging`.

    FullRiskManager.__init__(self, *, config: RiskConfig, store=None, symbol: str = "MES"):
    - `self._config = config`
    - `self._store = store`   (DuckDBStore instance; None in unit tests)
    - `self._instrument = get_instrument(symbol)`
    - `self._active_positions: set[str] = set()`  (strategy_ids with open positions)
    - `self._log = get_logger(__name__)`
    - `self._session_id: str = new_run_id()`

    FullRiskManager HWM state (maintained in-memory, written to DuckDB on each check):
    - `self._hwm_static: Decimal`
    - `self._hwm_trailing_eod: Decimal`
    - `self._hwm_trailing_intraday: Decimal`
    - Initialize from `config.account_equity` in `__init__`; Phase 5 startup bootstrap
      overrides these via `load_hwm_from_db(date_str, store)` called by the API lifespan.

    Method `async def check(self, signal: Signal, state: RiskState) -> RiskDecision`:
    1. Compute current_equity = config.account_equity + state.realized_pnl_today + state.open_exposure_dollars
    2. Update HWM for TRAILING_INTRADAY (always updates to max of current and prior HWM)
    3. Compute all three floors: floor_X = hwm_X - config.daily_dd_limit
    4. Check concurrency cap (RM-08): if signal.strategy_id in _active_positions → reject
    5. Check DD circuit breaker (RM-04): if realized_pnl + open_exposure <= -daily_dd_limit → reject
    6. Compute proposed_size = min(size_for_stop(risk_$, stop_distance_points, instrument), max_contracts)
       where risk_$ = config.account_equity * config.max_risk_per_trade_pct
       and stop_distance_points = abs(signal.entry - signal.stop)
    7. Check worst_case_loss (RM-03): compute worst_case = stop_distance_points * instrument.point_value * proposed_size
       active floor = floor based on config.drawdown_model
       if current_equity - worst_case < active_floor → reject with 'dd_floor_violation'
    8. Approved: return RiskDecision(approved=True, reason='approved', adjusted_size=proposed_size)
    9. Persist: if self._store is not None, call write_risk_state and write_audit_event synchronously

    Method `record_position_open(strategy_id: str) -> None`:
    - Add strategy_id to _active_positions

    Method `record_position_closed(strategy_id: str) -> None`:
    - Discard strategy_id from _active_positions (no-op if not present)

    Method `load_hwm_from_db(date_str: str, store) -> None`:
    - Called on startup to restore HWM from yesterday's last risk_state row (D-08)
    - If no row found (Day-1 exception): use config.account_equity as starting HWM

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| signal → check() | Signal values (entry, stop, size_hint) come from strategy; must be validated |
| check() → DuckDB | Write path; failure must not silently pass bad data |
| _active_positions set | In-memory only; lost on process restart — acceptable for v1 paper mode |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-02-01 | Spoofing | signal.strategy_id | accept | Single-operator local; strategy_id is set by trusted strategy code, not user input |
| T-05-02-02 | Tampering | size_for_stop arithmetic | mitigate | Pure function with Decimal-only arithmetic; unit tests lock the exact output for (1000, 5, MES/ES) |
| T-05-02-03 | Repudiation | RiskDecision not persisted | mitigate | Every check() result written to audit_log with reason_code; write happens before coroutine yields |
| T-05-02-04 | Denial of Service | DB write in hot path | accept | DuckDB writes at 1m bar rates are < 1ms; paper mode has no performance SLA |
| T-05-02-05 | Elevation of Privilege | concurrency_cap bypass | mitigate | _active_positions checked atomically in check(); asyncio single-threaded so no race |
</threat_model>
