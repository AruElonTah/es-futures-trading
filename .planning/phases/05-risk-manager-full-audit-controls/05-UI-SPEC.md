# Phase 5 — UI Design Contract

> Visual and interaction contract for Phase 5: Risk Manager + Full Audit + Controls.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Source authority:** All design decisions pre-populated from existing Phase 3/4 codebase
> patterns. No new design decisions were required; this contract locks the extension of the
> established Bloomberg-terminal dark-monospace theme into two new surfaces: the blotter
> sub-route and the hotkey registry overlay.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (vanilla Tailwind v3.4.19 + inline styles) |
| Icon library | none (text labels and Unicode glyphs only) |
| Font | CSS system monospace stack via `font-mono` Tailwind class (body `className="font-mono"` on root layout) |

**No shadcn:** This project does not use a component library. All UI is hand-authored with
Tailwind v3 utility classes and inline `style` props following the pattern established in
Phase 3 (`dashboard/page.tsx`) and Phase 4 (`optimizations/page.tsx`). This is the only
approach consistent with the existing codebase — do not introduce shadcn or any component
library in Phase 5.

## Spacing Scale

Declared values (multiples of 4, matching the inline-style tokens already in use across
`dashboard/page.tsx`, `optimizations/page.tsx`, and all existing components):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge internal padding, inline tight elements |
| sm | 8px | Row internal padding (blotter cells), button vertical padding, banner gap |
| md | 16px | Header horizontal padding, section horizontal padding, element spacing |
| lg | 24px | Section vertical gaps, table section margins |
| xl | 32px | Empty-state vertical padding |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Not used in Phase 5 surfaces |

Exceptions:
- Header height: 48px (established in Phase 3; not a spacing token — it is a fixed layout dimension)
- Blotter table row height: 36px (compact data rows matching terminal density; not a spacing token)
- Modal overlay: 100vw × 100vh fixed, with inner dialog max-width 480px
- Confirmation input: full-width inside the dialog with 8px vertical + 12px horizontal padding

## Typography

All text is monospace. The root `<body>` has `className="font-mono"` from `layout.tsx`.
Tailwind v3 maps `font-mono` to the OS system monospace stack.
Four sizes in active use across Phase 3/4 (locked — Phase 5 uses the same set):

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / cell data | 12px | 400 (regular) | 1.4 |
| Label / column header | 11px | 400 (regular) | 1.3 |
| Section header / nav link | 14px | 700 (bold) | 1.2 |
| Display / system title | 14px (same as section header) | 700 (bold) | 1.2 |

**Rule:** Only two weights are used — 400 (regular) and 700 (bold). The `font-bold` Tailwind
class maps to 700. No 500, 600, or 800 weights appear in this codebase.

**Tabular numerics:** All P&L, price, tick, and time values use `tabular-nums` CSS feature
(`className="tabular-nums"`) to prevent column-width jitter on live updates. This is an
established pattern from `ETClock.tsx`.

### 1. `/dashboard/blotter/page.tsx` — Blotter Sub-Route

**Layout:** Full-height column (same structure as `dashboard/page.tsx`).
- Header: 48px fixed height, same structure as existing dashboard header.
  - Left: `← Dashboard` link (`color: #888`, `fontSize: 12px`) matching optimizations page back-link pattern.
  - Center: `BLOTTER` title (`color: #d1d4dc`, `fontWeight: bold`, `fontSize: 14px`).
  - Right: Engine state badge (see Engine State Badge below) + ET clock + connection status.
- Body: Full remaining height, `overflow: auto`, positions table filling the space.

**Positions Table:**
```
Column order (left → right):
  Symbol | Side | Qty | Avg Fill | Mark | Unreal P&L | Stop Dist $ | Stop Dist ticks | Target Dist | Time In
```
- `symbol`: `#d1d4dc`, 12px
- `side`: `LONG` in `#4ade80`, `SHORT` in `#f87171`
- `qty`: `#d1d4dc`, right-aligned, tabular-nums
- `avg_fill`: `#d1d4dc`, right-aligned, tabular-nums, 2 decimal places
- `mark`: `#d1d4dc`, right-aligned, tabular-nums, 2 decimal places; updates on each WS `bar_received`
- `unreal_pnl`: positive → `#4ade80`; negative → `#f87171`; zero → `#888`. Right-aligned, tabular-nums, always show sign (+/-), 2 decimal places, dollar symbol prefix
- `stop_dist_$`: `#f87171`, right-aligned, tabular-nums; negative number (distance to stop in dollars)
- `stop_dist_ticks`: `#888`, right-aligned, tabular-nums
- `target_dist`: `#4ade80`, right-aligned, tabular-nums
- `time_in`: `#888`, right-aligned; format `HH:MM:SS` counting up from entry fill

Column headers: `fontSize: 11px`, `color: #888`, `textTransform: uppercase`, `letterSpacing: 0.05em` — matching the existing optimizations-page section label pattern.

Table borders: 1px `#222` bottom on header row; 1px `#1a1a2e` between data rows.

**Controls row (below table, 48px fixed):**
- Left side: `F — Flatten All` button and `K — Kill Switch` button
- Right side: `P — Pause` button
- All buttons: `fontSize: 12px`, `fontFamily: monospace`, `padding: 4px 12px`, `borderRadius: 4px`
- Flatten button: `border: 1px solid #ef4444`, `color: #ef4444`, `backgroundColor: transparent`
- Kill button: `border: 1px solid #dc2626`, `color: #dc2626`, `backgroundColor: transparent`
- Pause button: `border: 1px solid #444`, `color: #888` (idle); `border: 1px solid #eab308`, `color: #eab308` (when engine is paused)

## Interaction Contracts

### Hotkey Registry

All four hotkeys are registered in a **single client-side registry** in
`apps/web/hooks/useHotkeys.ts` (new hook). The registry is the single source of truth —
the `HelpOverlay` reads from it to render the table. Hotkey collision detection throws at
hook initialization, not at runtime.

| Key | Trigger condition | Action |
|-----|------------------|--------|
| `F` | Any page within `/dashboard/*`; not inside an `<input>` or `<textarea>` | Open Flatten confirmation dialog |
| `K` | Any page within `/dashboard/*`; not inside an `<input>` or `<textarea>` | Open Kill confirmation dialog |
| `P` | Any page within `/dashboard/*`; not inside an `<input>` or `<textarea>` | Call `POST /pause` (toggle); update engine state badge |
| `?` | Any page within `/dashboard/*` (including inside inputs — use `keydown` without `target.tagName` guard for `?`) | Toggle help overlay |

**Guard rule for F, K, P:** Skip handler when `document.activeElement.tagName` is `INPUT`,
`TEXTAREA`, or `SELECT`. This prevents the hotkeys from firing while the user is typing the
"FLATTEN" / "KILL" confirmation string.

**Re-entrant guard:** If a confirmation dialog is already open, pressing `K` while the
Flatten dialog is open (or vice versa) has no effect — the existing dialog stays open.

### Blotter Live-Update Contract

The blotter page subscribes to `WS /stream` via the existing `useStream` hook. On each
`bar_received` event, the mark price for every open position is updated to the bar's `close`
value. Unrealized P&L recalculates client-side:
`unreal_pnl = (mark - avg_fill) × qty × tick_value / tick_size`
(for long positions; sign-flipped for short). This matches CONTEXT.md Specifics.

Polling fallback: `GET /positions` via TanStack Query with `refetchInterval: 1000` as a
fallback when WS is disconnected.

### Confirmation Dialog Focus Contract

On dialog open:
1. Focus is trapped inside the dialog (`Tab` cycles only within the dialog).
2. The text input receives focus automatically (`autoFocus` attribute).
3. `Escape` closes the dialog and focuses the triggering button (or body if triggered by hotkey).

On confirm:
1. The button shows no loading spinner (operations complete quickly in paper mode).
2. The dialog closes immediately on confirm click / Enter keypress.
3. The engine state badge in the blotter header updates within one WS event cycle.

### Engine State Persistence Contract

Kill state persists across page reloads. On blotter page mount, `GET /positions` response
includes `engine_state` field. The Zustand store (`store/ws.ts`) is extended with an
`engineState: 'running' | 'paused' | 'killed'` field, initialized from the API response.
WS `engine_state_changed` events update this field in real time.

## Accessibility Contract

- All interactive elements have `role` and `aria-label` attributes where implicit semantics are insufficient.
- Confirmation dialogs use `role="dialog"`, `aria-modal="true"`, `aria-labelledby` pointing to the title element.
- Help overlay uses `role="dialog"`, `aria-modal="true"`.
- Disabled confirm buttons carry `aria-disabled="true"` (not just the `disabled` attribute) so screen readers announce the state.
- The blotter table uses `<table>`, `<thead>`, `<tbody>`, `<th scope="col">` — semantic HTML, not `div`-based grid.
- Color alone is never the sole differentiator for P&L sign — positive values show `+` prefix, negative values show `-` prefix (sign character).
