    5. await bus.publish(TOPIC_ENGINE_STATE, {"type": "engine_state_changed", "payload": {"state": new_state}})
    6. return {"state": new_state}

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| POST /kill, /flatten, /pause | No auth in v1 (single operator localhost); all three mutate engine state |
| HWM bootstrap from DuckDB | Read-only query; no injection surface (date is computed from system clock) |
| EodScheduler wall clock | Depends on system TZ and ZoneInfo; DST transitions handled by zoneinfo library |
| kill-switch Event bootstrap | get_engine_state() read on startup; trusted local DuckDB |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-04-01 | Spoofing | POST /kill, /flatten without auth | accept | Single-operator localhost; CORS restricted to localhost:3000; no auth surface in v1 |
| T-05-04-02 | Repudiation | kill/flatten not audited | mitigate | Both endpoints write to audit_log via write_audit_event before returning |
| T-05-04-03 | Denial of Service | EodScheduler asyncio.sleep | accept | Sleep target is wall-clock; missed fire (process down) is an accepted risk for paper mode |
| T-05-04-04 | Tampering | CORS method expansion to POST | accept | T-03-05-02 threat model was GET-only; expanding to POST on localhost is acceptable; restrict to specific routes in Phase 7 if needed |
| T-05-04-05 | Information Disclosure | GET /positions leaks position data | accept | Single-operator localhost; CORS restricted; no sensitive data beyond paper fills |
| T-05-04-06 | Denial of Service | kill-switch Event not restored on restart | mitigate | load_kill_state_from_db() called on lifespan startup bootstraps asyncio.Event from DuckDB (D-10) |
